# Professor Pixel's Programming Palace — Agent Entry Point

## Project Identity

**Name:** Professor Pixel's Programming Palace (`professor-pixel-programming-palace`)
**Purpose:** A dual-language (Python + JavaScript) programming education platform for kids, featuring
a conversational mascot (Pixel) who guides learners through building real games.
**Stack:** pnpm monorepo · React 19 · Vite 6 · TypeScript 5.9 · Tailwind CSS 4 · Babylon.js 8 ·
Pyodide · Biome · Playwright · Vitest

## Monorepo Layout

```
apps/
  web/          — React 19 SPA (@professor-pixel/web)
  mobile/       — React Native + Expo (@professor-pixel/mobile, stub)
packages/
  types/        — Zod schemas and shared TypeScript types
  config/       — Shared constants
  core/         — Zustand stores and runtime utilities
  python-sandbox/ — Pyodide wrapper for in-browser Python execution
  lesson-engine/  — Lesson orchestration, JS sandbox, conversion utilities
shared/         — Legacy shared schema (TypeScript interfaces)
assets/         — Game assets (sprites, audio, fonts) tracked by Git LFS
tests/          — Shared test utilities and Playwright E2E suite
```

## Key Commands

```bash
# Install (pnpm required)
pnpm install

# Start web app dev server (http://localhost:5173)
pnpm dev

# Type-check all packages
pnpm check

# Lint and format with Biome
pnpm lint
pnpm format

# Run unit tests (Vitest)
pnpm test

# Run E2E tests (Playwright) — requires dev server running
npx playwright test

# Run critical E2E suite only
npx tsx tests/e2e/run-comprehensive-tests.ts --critical

# Build all packages
pnpm build
```

## Architecture Constraints

- Max 300 LOC per file — enforced globally.
- No Strata (`@jbcom/strata`), no React Three Fiber, no Astro — these were removed.
- Use Biome for all linting and formatting (not ESLint/Prettier).
- Workspace dependencies use `workspace:*` protocol.
- Python execution is 100% client-side via Pyodide WebAssembly — no server needed.
- Static mode auto-detects GitHub Pages (`VITE_STATIC_MODE=true` or `*.github.io` hostname).

## Package Dependency Order

```
types → config → core → python-sandbox, lesson-engine → apps/web, apps/mobile
```

## Testing Approach

- Unit tests: Vitest in each package and the web app.
- E2E: Playwright in `tests/e2e/`, targeting `localhost:5173`.
- Multi-resolution: desktop 1920×1080 through mobile 375×667.
- Error detection: runtime JS errors, Vite overlay, network failures.

## Current Known Issues

- Monaco editor integration is incomplete; code editors use a basic textarea fallback.
- Mobile app (`apps/mobile`) is a stub — not functional.
